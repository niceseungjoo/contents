# 머신러닝 기본 흐름 이해

이번주 주제는 머신러닝 프로젝트의 기본적인 수행 흐름과 핵심 개념을 정리하는 것을 목적으로 한다. 머신러닝 모델 개발은 크게 데이터 준비, 모델 학습, 성능 평가의 단계로 구성된다. 각 단계의 명확한 이해는 효과적이고 신뢰성 있는 모델을 구축하는 기반이 된다.

1. 머신러닝의 기본 흐름

머신러닝 프로젝트는 일반적으로 데이터셋 확보, 데이터 전처리, 모델 학습, 모델 평가의 순차적인 흐름을 따른다.
데이터셋 → 전처리 → 학습 → 평가

1.1. 데이터셋 (Dataset)
머신러닝 모델을 구축하기 위한 기초 자산으로, 해결하고자 하는 문제와 관련된 정보의 집합이다. 데이터의 양과 질이 모델의 최종 성능을 결정하는 핵심 요소로 작용한다.

1.2. 데이터 전처리 (Data Preprocessing)
수집된 원시 데이터(Raw Data)를 모델이 학습하기에 적합한 형태로 가공하는 과정이다. 데이터의 품질을 향상시켜 모델의 학습 효율과 성능을 높이는 것을 목표로 하며, 주요 작업은 다음과 같다.

• 결측치 처리 (Missing Value Imputation): 누락된 데이터를 특정 값(평균, 중앙값 등)으로 대체하거나 해당 데이터를 제거한다.
• 이상치 탐지 및 제거 (Outlier Detection and Removal): 정상적인 데이터 분포에서 크게 벗어난 값을 식별하고 처리한다.
• 데이터 정규화 또는 표준화 (Normalization/Standardization): 각 특성(Feature)의 데이터 범위를 일치시키거나 분포를 조정하여 모델이 특정 특성에 과도하게 영향을 받는 것을 방지한다.

1.3. 모델 학습 (Model Training)
전처리가 완료된 데이터를 사용하여 모델이 데이터 내의 패턴이나 규칙을 스스로 발견하도록 하는 과정이다. 이 단계에서 모델은 주어진 입력(Feature)과 정답(Label) 간의 관계를 학습한다.

1.4. 모델 평가 (Model Evaluation)
학습된 모델이 새로운 데이터에 대해 얼마나 정확한 예측을 수행하는지 성능을 측정하는 단계이다. 객관적인 지표를 통해 모델의 일반화 성능과 실효성을 검증한다.

2. 주요 개념 정의
2.1. Feature / Label
• Feature (특성): 모델의 입력 변수로, 예측의 근거가 되는 데이터의 개별 속성을 의미한다. 예를 들어, 주택 가격 예측 모델에서 '면적', '방의 개수', '건축 연도' 등은 Feature에 해당한다.
• Label (레이블): 모델이 예측해야 할 출력 변수, 즉 정답 값을 의미한다. 위 예시에서 '주택 가격'이 Label이 된다.

2.2. Train/Test Split (훈련/테스트 데이터 분할)
보유한 전체 데이터셋을 모델 학습용과 평가용으로 분리하는 과정이다.
• Train Set (훈련 데이터): 모델을 직접 학습시키는 데 사용되는 데이터 집합이다.
• Test Set (테스트 데이터): 학습에 사용되지 않은 데이터로, 오직 모델의 최종 성능을 평가하는 데만 사용된다.

데이터를 분할하는 주된 목적은 모델이 학습 데이터에만 과도하게 최적화되는 **과적합(Overfitting)**을 방지하고, 새로운 데이터에 대한 일반화 성능을 객관적으로 측정하기 위함이다. 일반적으로 전체 데이터의 70~80%를 Train Set으로, 20~30%를 Test Set으로 분할한다.
